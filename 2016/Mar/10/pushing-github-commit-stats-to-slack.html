Pushing github commit stats to slack

It all started with showing metrics. What every dev work on last day? How are they performing and so on. We are heavily dependent on github for code management. 50 commits everyday is an average. Except the regular planned task, we were not able to get much insights on where developers spend time. Because github is the final destination to every single task, so we thought of measuring that.

We wrote a script to fetch all last day commits from github and push them to slack #dev channel. Every morning 6 AM it pushes all the commits by person and repository.

Where this help?

  • To see what all happened in engineering last day
  • What all repository got updates. This is really helpful to see where the focus going on.
  • To encourage devs to do more commits ;)
